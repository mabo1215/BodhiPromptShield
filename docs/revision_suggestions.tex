\subsection*{三、当前最适合的投稿定位}
按现在这篇稿子的形态，我建议你把它定位成：

\begin{itemize}
    \item \textbf{practical systems/privacy paper}；
    \item \textbf{security/privacy + LLM agent deployment} 交叉方向；
    \item \textbf{NLP/LLM application paper}，但不是纯 benchmark paper，也不是纯 theorem/cryptography paper。
\end{itemize}

因此最适合的 venue 不是那种特别偏理论安全、也不是那种必须有超大公开 benchmark 的顶会主轨，而是：
\begin{itemize}
    \item 接受系统型、部署型、测评型安全论文的会议；
    \item 或接受应用导向 AI/privacy/security 论文的期刊。
\end{itemize}

\subsection*{四、我最推荐的会议}
下面按“\textbf{适配度 + 现实可投性}”排序给你建议。

\subsubsection*{1. ACM CCS 2026}
\textbf{推荐度：最高（如果你想冲更强会议）}

\paragraph{为什么适合}
你的论文现在已经不只是 prompt engineering，而是：
\begin{itemize}
    \item 有明确 threat model；
    \item 有 interface-layer mediation architecture；
    \item 有 agent propagation 风险；
    \item 有 restoration boundary；
    \item 有 multimodal/OCR exposure；
    \item 有一整套 controlled evaluation。
\end{itemize}
这套东西和 CCS 的“real-world security/privacy systems”口味是能对上的，尤其如果你把 framing 再略微往 security systems 靠一点，会更合适。CCS 2026 官方 CFP 明确面向计算机安全与隐私的真实问题，2026 有两个 review cycle；第二个 cycle 的摘要截止为 \textbf{2026-04-22}，全文截止为 \textbf{2026-04-29}，官方投稿/CFP 页面见引文。:contentReference[oaicite:3]{index=3}

\paragraph{风险}
\begin{itemize}
    \item 竞争会非常强；
    \item 你的工作更偏 deployment framework + controlled evaluation，而不是攻防新原语；
    \item 如果想投 CCS，建议在标题、摘要、Introduction 中再强化 ``agent propagation risk''、``pre-inference security boundary'' 和 ``deployment threat surface''。
\end{itemize}

\paragraph{结论}
\textbf{如果你想冲高水平安全会议，CCS 是现在最值得尝试的。} 投稿入口与最终 deadline 见官方页面引文。:contentReference[oaicite:4]{index=4}

\subsubsection*{2. RAID 2026}
\textbf{推荐度：很高（比 CCS 更现实）}

\paragraph{为什么适合}
RAID 比较欢迎：
\begin{itemize}
    \item 新的攻击面和威胁面；
    \item 防御/检测/缓解；
    \item measurement papers；
    \item tools/datasets/experience papers。
\end{itemize}
你的论文如果强调：
\begin{itemize}
    \item prompt exposure surface；
    \item propagation across retrieval/memory/tools；
    \item pre-inference mitigation；
    \item controlled benchmark and middleware design；
\end{itemize}
其实和 RAID 的口味非常接近。RAID 2026 官方 CFP 给出的 submission deadline 是 \textbf{2026-04-16 AoE}，官网/投稿信息见官方 CFP。:contentReference[oaicite:5]{index=5}

\paragraph{风险}
\begin{itemize}
    \item RAID 更偏 security research community；
    \item 你需要在摘要和实验讨论里稍微强调“attacks, intrusions, defenses”视角，而不只是 utility preservation。
\end{itemize}

\paragraph{结论}
\textbf{如果你想投 security venue 且希望命中率比 CCS 更现实，RAID 是我非常推荐的选择。} 官方投稿页见引文。:contentReference[oaicite:6]{index=6}

\subsubsection*{3. EMNLP 2026}
\textbf{推荐度：高（如果你更想走 NLP/LLM application 路线）}

\paragraph{为什么适合}
你的论文也完全可以被包装成：
\begin{itemize}
    \item LLM prompt mediation；
    \item privacy-aware prompting；
    \item utility-preserving prompt transformation；
    \item multimodal prompt sanitization。
\end{itemize}
EMNLP 2026 官方 CFP 说明主会投稿采用 ARR 机制：\textbf{ARR submission deadline 为 2026-05-25}，\textbf{EMNLP commitment deadline 为 2026-08-02}，会期为 2026 年 10 月下旬，官方页面见引文。:contentReference[oaicite:7]{index=7}

\paragraph{风险}
\begin{itemize}
    \item 如果投 EMNLP，你要把 ``security/privacy threat model'' 的表述稍微降一点，把 ``NLP utility, prompting, agent interaction'' 的表述再抬高一点；
    \item 当前稿子对 security venue 更天然，对纯 NLP venue 稍微需要再做 wording adaptation。
\end{itemize}

\paragraph{结论}
\textbf{如果你更希望走 NLP/LLM 主流社区，而不是 security 社区，EMNLP 2026 是当前最合适的会议入口。} 官方投稿/commit 页面见引文。:contentReference[oaicite:8]{index=8}

\subsection*{五、不太建议优先投的会议}
\begin{itemize}
    \item \textbf{纯理论安全会议：} 你的工作不是 cryptographic guarantee、formal proof、privacy theorem 取向。
    \item \textbf{纯 benchmark/data 会议：} 你的 CPPB 是 controlled benchmark，不是大规模社区 benchmark。
    \item \textbf{纯对话系统/agent UX 会议：} 你的 paper 核心不是人机交互体验，而是 privacy mediation under threat model。
\end{itemize}

\subsubsection*{1. Computers \& Security}
\textbf{推荐度：最高}

\paragraph{为什么适合}
这是我对你当前稿子最推荐的 journal。  
它的官方页面明确面向 IT security research 和 practical applications，强调既有研究创新也有实践价值；而你的稿子正是 \textbf{practical privacy/security system for LLM/VLM agents} 这一路线。官方页面还提供投稿入口，且该刊为常规期刊模式，\textbf{rolling submission}。:contentReference[oaicite:9]{index=9}

\paragraph{为什么比顶刊更适合}
\begin{itemize}
    \item 你的贡献更偏 deployment framework 和 evaluation；
    \item 不是特别强的 theorem paper；
    \item 不是必须要极大公开 benchmark 才能成立的 paper；
    \item 但非常适合 security applications / systems 期刊。
\end{itemize}

\paragraph{结论}
\textbf{如果你现在就想稳妥投稿，Computers \& Security 是第一推荐。} 官方投稿页见引文。:contentReference[oaicite:10]{index=10}

\subsubsection*{2. IEEE Transactions on Dependable and Secure Computing (TDSC)}
\textbf{推荐度：高}

\paragraph{为什么适合}
TDSC 官方 scope 明确包括 secure systems、evaluation、modeling、methodologies and mechanisms for dependable and secure systems。你的 paper 从系统与安全边界视角看，是能挂上 TDSC 的，尤其是：
\begin{itemize}
    \item secure middleware；
    \item system-level privacy exposure reduction；
    \item evaluation of secure/dependable behavior；
    \item agent pipeline propagation control。
\end{itemize}
TDSC 官方 CFP/Topics 页见引文；该刊为常规期刊，\textbf{rolling submission}。:contentReference[oaicite:11]{index=11}

\paragraph{风险}
\begin{itemize}
    \item 相比 Computers \& Security，TDSC 会更看重系统与安全方法论的“硬度”；
    \item 如果投它，最好再把 propagation/security architecture 的 framing 提高一点。
\end{itemize}

\paragraph{结论}
\textbf{如果你想冲更强一些的安全期刊，TDSC 值得考虑。} 官方页面见引文。:contentReference[oaicite:12]{index=12}

\subsubsection*{3. IEEE Transactions on Information Forensics and Security (TIFS)}
\textbf{推荐度：中高}

\paragraph{为什么适合}
TIFS 官方 scope 覆盖 information forensics, information security, systems applications。你的工作和 ``information security + systems applications'' 这部分是有关联的，尤其是 privacy-sensitive prompt protection。官方页面说明新投稿通过 ScholarOne，且鼓励 reproducible research。:contentReference[oaicite:13]{index=13}

\paragraph{风险}
\begin{itemize}
    \item TIFS 往往更喜欢更“硬”的安全/取证/识别/检测问题；
    \item 你的稿子偏系统部署与 prompt mediation，和 TIFS 的最核心口味不如 TDSC/Computers \& Security 那么贴。
\end{itemize}

\paragraph{结论}
\textbf{可以投，但不是第一优先。} 如果你想冲更强 IEEE 期刊，可以尝试。:contentReference[oaicite:14]{index=14}

\subsubsection*{4. Cybersecurity (Springer)}
\textbf{推荐度：中高，比较稳}

\paragraph{为什么适合}
Springer 的 \emph{Cybersecurity} 官方 aims and scope 明确包括 software/system security、privacy-enhancing technologies、AI security 等。你的工作作为 AI security / privacy-oriented deployment paper，比较容易放进去，且期刊常规模式通常也是 \textbf{rolling submission}。:contentReference[oaicite:15]{index=15}

\paragraph{结论}
\textbf{如果你想找一个更稳、而且 scope 对 AI security/privacy 很友好的期刊，Cybersecurity 是不错的备选。} 官方 scope 见引文。:contentReference[oaicite:16]{index=16}

\subsubsection*{5. ACM Transactions on AI Security and Privacy (TAISAP)}
\textbf{推荐度：中高，但更偏“新刊机会”}

\paragraph{为什么适合}
ACM 2025 年宣布了新刊 \emph{ACM Transactions on AI Security and Privacy (TAISAP)}，其范围明确覆盖 AI 模型和 AI-enabled systems 的 security/privacy 以及 deployment-level AI security questions。你的 paper 主题与它非常对口。ACM 官方还说明该刊在 2026--2029 前三年 open access 且无 APC。:contentReference[oaicite:17]{index=17}

\paragraph{风险}
\begin{itemize}
    \item 新刊，影响力与审稿风格还在形成期；
    \item 如果你更看重传统期刊稳妥性，Computers \& Security / TDSC 会更稳。
\end{itemize}

\paragraph{结论}
\textbf{如果你愿意押一个方向非常匹配的新刊，TAISAP 值得认真考虑。} 官方说明见引文。:contentReference[oaicite:18]{index=18}
