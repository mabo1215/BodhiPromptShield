\subsection*{New Section IV-A: System Overview}
Insert the following text as the starting point of the new method section:

\begin{verbatim}
\subsection{System Overview}

The proposed framework acts as a privacy mediation layer between the user and a downstream LLM/VLM-based agent. Instead of directly sending the raw prompt $x$ to the model, the framework first analyzes the input for privacy-sensitive content, transforms the identified spans into protected surrogates, and then forwards a sanitized prompt $\tilde{x}$ to the downstream inference pipeline.

The full workflow contains four stages:
\begin{enumerate}
    \item privacy-sensitive span extraction;
    \item semantic-preserving prompt sanitization;
    \item downstream agent inference on the sanitized prompt;
    \item optional secure restoration of selected entities under access control.
\end{enumerate}

This design allows the framework to reduce privacy exposure before model calls, retrieval, tool invocation, or logging, while preserving the semantic intent necessary for useful downstream responses.
\end{verbatim}

\subsection*{New Section IV-B: Privacy Entity Extraction}
Insert the following text:

\begin{verbatim}
\subsection{Privacy Entity Extraction}

Given an input prompt $x$, we first identify candidate privacy-sensitive spans:
\[
E(x)=\{(s_i,c_i,p_i)\}_{i=1}^{n},
\]
where $s_i$ denotes a detected span, $c_i$ is its privacy category, and $p_i \in [0,1]$ is a confidence score.

The detector may combine multiple signals:
\begin{itemize}
    \item rule-based recognizers for structured entities such as email addresses, phone numbers, account numbers, IDs, dates of birth, and addresses;
    \item named entity recognition for names, organizations, locations, and domain-specific entities;
    \item LLM-assisted contextual privacy judgment for spans whose sensitivity depends on surrounding semantics;
    \item optional OCR or VLM-based extraction when privacy-bearing content appears in images or screenshots.
\end{itemize}

A span is marked for protection when its confidence exceeds a configurable threshold $\tau$ or when it matches a high-risk privacy policy. The output of this stage is a structured privacy annotation over the prompt.
\end{verbatim}

\subsection*{New Section IV-C: Semantic-Preserving Prompt Sanitization}
Insert the following text:

\begin{verbatim}
\subsection{Semantic-Preserving Prompt Sanitization}

After privacy-sensitive spans are detected, the framework transforms them into protected surrogates through a sanitization operator
\[
T(s_i,c_i)\rightarrow \hat{s}_i.
\]
The sanitized prompt is then
\[
\tilde{x}=T(x)=x[s_1 \mapsto \hat{s}_1,\ldots,s_n \mapsto \hat{s}_n].
\]

We consider three main sanitization modes:
\begin{enumerate}
    \item \textbf{Typed placeholder replacement}, such as replacing a real person name with \texttt{<PERSON\_1>};
    \item \textbf{Semantic abstraction}, such as replacing a full address with ``a residential address in Auckland'';
    \item \textbf{Secure symbolic mapping}, where protected spans are replaced by reversible secure tokens linked to a local mapping table.
\end{enumerate}

The choice of sanitization mode depends on the privacy category, downstream task requirement, and whether later restoration is necessary. The key objective is to reduce exposure risk while preserving enough semantic structure for correct downstream reasoning.
\end{verbatim}

\subsection*{New Section IV-D: Utility-Constrained Prompt Mediation}
Insert the following text:

\begin{verbatim}
\subsection{Utility-Constrained Prompt Mediation}

Prompt protection should not destroy the user's original intent. We therefore formulate the mediation stage as a privacy--utility trade-off:
\[
\min_{T}\ \mathcal{L}_{privacy}(x,\tilde{x})+\lambda \mathcal{L}_{utility}(f(x),f(\tilde{x})),
\]
where $\mathcal{L}_{privacy}$ measures residual exposure after sanitization, $\mathcal{L}_{utility}$ measures task or semantic degradation, and $\lambda$ controls the privacy--utility balance.

In practice, $\mathcal{L}_{privacy}$ can be approximated using exposure rate, recoverability, or redaction success, while $\mathcal{L}_{utility}$ can be measured using semantic similarity, answer consistency, task success rate, or agent execution correctness.

This formulation emphasizes that the goal of the framework is not simply to mask tokens, but to preserve usable intent after protection.
\end{verbatim}

\subsection*{New Section IV-E: Agent-Level Integration}
Insert the following text:

\begin{verbatim}
\subsection{Agent-Level Integration}

The framework is designed for deployment in agent pipelines rather than only standalone prompting. Let the downstream system be represented as
\[
\mathcal{A}=\{f_{\mathrm{LLM/VLM}},\mathcal{R},\mathcal{T},\mathcal{M}\},
\]
where $f_{\mathrm{LLM/VLM}}$ is the foundation model, $\mathcal{R}$ is an optional retrieval module, $\mathcal{T}$ is the toolset, and $\mathcal{M}$ is memory or logging infrastructure.

Without protection, raw prompts may propagate through all of these components. In the proposed design, the middleware intercepts the user input before downstream execution and only releases the sanitized prompt $\tilde{x}$. This reduces direct exposure of sensitive content in model calls, retrieval queries, memory traces, and tool arguments.

Authorized restoration may optionally occur after the model returns its output, depending on system policy and task requirement.
\end{verbatim}

\subsection{Extension to VLM-Based Multimodal Inputs}

For multimodal settings, the user input may contain both text and images:
\[
x^{(m)}=(x^{(t)},x^{(v)}).

Privacy-sensitive information may appear not only in textual prompts, but also in screenshots, scanned documents, medical reports, invoices, receipts, identity documents, whiteboards, and natural-scene text.

The framework first extracts candidate privacy-bearing visual content using OCR, visual grounding, or VLM-assisted detection, and then applies the same sanitization logic to generate a protected multimodal representation
\[
\tilde{x}^{(m)}=(\tilde{x}^{(t)},\tilde{x}^{(v)}).
\]

This enables privacy-preserving inference for multimodal question answering, visual document understanding, and image-assisted agent systems.
\end{verbatim}

\subsection{Evaluation Metrics}

We report the following metrics:
\begin{itemize}
    \item Sensitive Span Precision / Recall / F1;
    \item Privacy Exposure Rate (PER);
    \item Semantic Similarity between original and sanitized prompts;
    \item Answer Consistency between outputs generated from raw and protected prompts;
    \item Task Success Rate for downstream agent execution;
    \item Latency Overhead introduced by the privacy middleware.
\end{itemize}

Privacy Exposure Rate is defined as
\[
\mathrm{PER}=\frac{\#\text{exposed sensitive tokens after sanitization}}{\#\text{sensitive tokens in the original prompt}}.
\]

A utility preservation ratio may also be reported as
\[
\mathrm{UPR}=\frac{\text{task score on sanitized prompt}}{\text{task score on original prompt}}.
\]

\section{Limitations}

Several limitations remain in the current framework. First, privacy sensitivity is context-dependent, and some spans cannot be reliably classified as sensitive without deeper task or user-specific policy knowledge. Second, utility preservation may degrade when exact identifiers are essential for downstream reasoning or tool execution. Third, multimodal privacy extraction remains challenging in images with dense text, handwriting, or weak OCR quality. Fourth, the framework currently prioritizes practical deployment and does not provide a formal cryptographic guarantee against all adversaries. Finally, benchmark construction for prompt privacy is still immature, and broader evaluation across domains is needed.

\section{Conclusion}

This paper reformulates privacy-preserving text processing as a real-time prompt privacy problem for LLM/VLM-based agents. Instead of focusing on classical text classification or training-time privacy alone, the proposed framework protects privacy at the interaction interface before downstream inference. By combining privacy-sensitive span extraction, semantic-preserving prompt sanitization, and agent-level mediation, the framework reduces privacy exposure while preserving downstream response utility.

The proposed approach is model-agnostic, supports practical deployment as a middleware layer, and can be extended to multimodal settings in which privacy-bearing information appears in both text and images. This establishes prompt-level privacy mediation as a practical direction for privacy-aware large-model systems.

Future work includes stronger benchmark design, richer multimodal privacy grounding, adaptive policy learning, tighter privacy--utility analysis, and broader evaluation in real-world agent workflows.
