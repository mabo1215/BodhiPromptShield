\section*{Top-Tier Section-by-Section Revision Checklist for the Revised Manuscript}

\subsection*{Overall Assessment}
The manuscript has improved substantially and is now much closer to a serious submission. In particular, the revised version:
\begin{itemize}
    \item moves conceptual/illustrative material out of the empirical foreground;
    \item reframes CPPB as a controlled benchmark rather than an implied public benchmark;
    \item elevates propagation suppression to a central systems result;
    \item includes a stronger deployment-style comparator via enterprise staged redaction;
    \item calibrates claims more carefully in terms of privacy scope, reproducibility, and external validity.
\end{itemize}

However, the paper is still \textbf{not fully top-tier ready}. The remaining weaknesses are now concentrated in:
\begin{enumerate}
    \item incomplete benchmark accounting for CPPB,
    \item the lack of at least one stronger external practical baseline,
    \item partial rather than end-to-end reproducibility for several reported results.
\end{enumerate}

The next revision should therefore focus on \textbf{hardening empirical credibility}, not only on polishing narrative structure.

\subsection*{Abstract}
\textbf{What is already improved:}
\begin{itemize}
    \item The abstract now clearly frames the problem as pre-inference mediation in LLM/VLM agents.
    \item The propagation result is explicitly stated.
    \item The conclusion is more calibrated and no longer sounds like a universal guarantee claim.
\end{itemize}

\textbf{What still needs revision:}
\begin{itemize}
    \item Move the propagation result even earlier in the abstract, ideally as the main validated claim rather than as a later supporting detail.
    \item Replace any remaining broad wording such as ``model-agnostic'' with more qualified language unless the exact backend scope is specified.
    \item Keep the controlled nature of CPPB explicit, but compress it more efficiently so that the abstract does not over-spend words on caveats.
    \item End on one sharp claim:
    \begin{quote}
        controlled systems evidence that pre-inference mediation reduces cross-stage propagation while preserving useful downstream behavior.
    \end{quote}
\end{itemize}

\textbf{Target abstract structure:}
\begin{enumerate}
    \item agent-pipeline privacy problem,
    \item why propagation matters,
    \item proposed pre-inference mediation framework,
    \item main propagation + privacy--utility findings,
    \item scope-calibrated concluding claim.
\end{enumerate}

\subsection*{Introduction}
\textbf{What is already improved:}
\begin{itemize}
    \item The introduction now frames propagation across logging, retrieval, memory, and tools as the central risk.
    \item The novelty is explicitly described as deployment-oriented rather than guarantee-oriented.
    \item The contribution list is more disciplined than in the earlier draft.
\end{itemize}

\textbf{What still needs revision:}
\begin{itemize}
    \item Further sharpen the contrast against enterprise redaction middleware and ordinary de-identification pipelines.
    \item Make the stage-of-protection distinction even more explicit:
    \begin{itemize}
        \item training-time privacy,
        \item stored-text de-identification,
        \item prompt security,
        \item pre-inference agent-boundary mediation.
    \end{itemize}
    \item Tighten the contribution list further so that every listed contribution maps to a directly validated experiment or a clearly labeled scope statement.
    \item Consider shortening the local-DP contribution wording if the optional span-level bound is not experimentally central in this paper.
\end{itemize}

\textbf{Top-tier goal for the introduction:}
The introduction should leave no ambiguity that the paper's main scientific question is:
\begin{quote}
Can pre-inference mediation suppress privacy propagation across agent boundaries without unacceptable degradation of downstream task behavior?
\end{quote}

    \item It better distinguishes this paper from training-time privacy and stored-text de-identification.
\end{itemize}

\textbf{What still needs revision:}
\begin{itemize}
    \item Add a sharper contrast paragraph focused specifically on why conventional enterprise redaction and clinical de-identification pipelines are insufficient once prompt fragments propagate into retrieval, memory, and tool arguments.

\subsection*{Experiments}
\textbf{What is already improved:}
\begin{itemize}
    \item CPPB is now explicitly presented as a controlled benchmark.
    \item The paired-comparison protocol is much clearer.
    \item The stronger enterprise staged redaction baseline is a meaningful improvement over the previous draft.
    \item The reproducibility scope is now much more transparent.
\end{itemize}

\textbf{What still needs revision:}
\begin{itemize}
    \item This section remains the biggest bottleneck for a top-tier submission.
    \item Add exact CPPB accounting:
    \begin{itemize}
        \item total prompt count,
        \item subset counts,
        \item privacy-category counts or percentages,
        \item prompt-source composition,
        \item number of templates and generated variants,
        \item text-only vs multimodal counts.
    \end{itemize}
    \item Add one stronger external practical baseline if at all feasible, for example:
    \begin{itemize}
        \item Microsoft Presidio-style pipeline,
        \item prompted LLM zero-shot de-identification,
        \item a stronger contextual redaction baseline.
    \end{itemize}
    \item Add a compact artifact map pointer in the main paper, not only in the appendix, so readers immediately know which results are code-backed and which remain controlled manuscript results.
    \item If possible, add a small repeated-run or sensitivity analysis for at least one key operating-point result so that the paper is not entirely deterministic-profile based.
\end{itemize}

\textbf{Top-tier goal for experiments:}
This section should convince a skeptical reviewer that CPPB is not merely a well-written benchmark idea, but a sufficiently specified and auditable controlled evaluation protocol.

privacy control is most effective before sensitive content propagates across agent boundaries.
\end{quote}

\subsection*{Conclusion}
\textbf{What is already improved:}
\begin{itemize}
    \item The conclusion is more cautious than before.
    \item It is better aligned with the controlled-evidence framing.
\end{itemize}

\textbf{What still needs revision:}
\begin{itemize}
    \item Make propagation suppression the single centerpiece of the concluding claim.
    \item Avoid ending on broad deployment language unless it is immediately qualified.
    \item End with one concrete next milestone, such as:
    \begin{itemize}
        \item full CPPB data-card release,
        \item stronger external baselines,
        \item broader real-world workflow validation.
    \end{itemize}
\end{itemize}

\textbf{Top-tier goal for conclusion:}
The conclusion should be short, memorable, and disciplined:
\begin{quote}
pre-inference mediation reduces cross-stage privacy propagation under controlled agent-style settings, while preserving useful downstream behavior, but stronger benchmark release and broader external validation remain necessary.
\end{quote}

The revised manuscript is now \textbf{materially stronger and closer to submission quality}. It is no longer mainly blocked by framing. Its remaining gap to a top-tier submission lies in \textbf{benchmark hardness, baseline strength, and reproducibility completeness}. If these final empirical issues are improved, the paper's case will become substantially more competitive.
