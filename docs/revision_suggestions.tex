\section{Introduction}

Large language models (LLMs) and vision-language models (VLMs) have become the core reasoning engines of modern intelligent assistants and agent systems. They are increasingly used in document analysis, software development, health-related consultation, enterprise workflows, multimodal search, and tool-augmented decision making. In such applications, users often provide prompts that contain sensitive or private information, including names, phone numbers, addresses, identifiers, financial content, medical descriptions, proprietary project details, and privacy-bearing text embedded in uploaded images or screenshots.

This creates an important practical privacy problem. In many real-world deployments, raw prompts are not only observed by the underlying model, but may also be exposed to logging systems, retrieval modules, external tools, audit pipelines, or third-party service providers. As a result, directly sending unprotected prompts to LLM- or VLM-based systems may lead to unnecessary privacy exposure even when the model itself is not explicitly malicious. In agent settings, the privacy risk may be amplified because sensitive content can propagate across multiple components during reasoning and tool invocation.

Existing privacy-preserving natural language processing research has mainly focused on training-time data protection, differentially private learning, secure model optimization, representation anonymization, or post hoc de-identification. While these directions remain important, they do not fully address the interface-level privacy risks that arise when users interact with deployed large-model systems through free-form prompts. In practice, privacy protection must often occur before inference, under real-time constraints, and without significantly degrading the semantic intent required for correct downstream reasoning.

This motivates a new research direction: prompt-level privacy protection for LLM/VLM-based agents. The central challenge is to identify privacy-sensitive spans in user prompts and transform them into protected forms that reduce exposure risk while preserving sufficient information for the downstream model to return accurate and useful responses. This problem is particularly challenging because sensitive content may appear in highly diverse forms, ranging from structured identifiers to context-dependent personal attributes, domain-specific confidential terms, and image-extracted text in multimodal scenarios.

To address this challenge, we reformulate the original privacy-preserving text processing problem into a real-time prompt mediation framework for large-model systems. Instead of treating privacy protection as a conventional text classification task, the revised framework introduces a privacy-aware middleware layer that operates between the user and the downstream LLM/VLM agent. This layer detects sensitive spans, sanitizes or replaces them using semantic-preserving transformations, and forwards the protected prompt for inference while aiming to preserve downstream task utility.

The main contributions of this paper are summarized as follows:
\begin{itemize}
    \item We redefine privacy-preserving text processing from a traditional classification-centered setting to a prompt-level protection problem for LLM/VLM-based agents.
    \item We propose a real-time privacy mediation framework that identifies sensitive spans, sanitizes prompts through semantic-preserving transformation, and supports controlled recovery through secure mapping.
    \item We introduce an agent-level deployment view in which privacy protection is performed before large-model inference, thereby reducing privacy leakage through model calls, logs, retrieval, and tool invocation.
    \item We extend the formulation toward multimodal settings in which privacy-bearing content may also appear in images, screenshots, or OCR-extracted text.
    \item We establish a new evaluation perspective centered on privacy leakage reduction and utility preservation rather than classical text classification accuracy alone.
\end{itemize}

The rest of this paper is organized as follows. Section II reviews related work. Section III defines the prompt privacy problem and threat model. Section IV presents the proposed framework. Section V describes the experimental setup. Section VI discusses the results. Section VII outlines limitations. Section VIII concludes the paper.

\section{Problem Definition and Threat Model}

\subsection{Problem Definition}

Let a user prompt be denoted by $x$, where $x$ may contain a set of privacy-sensitive spans
\[
S(x) = \{s_1, s_2, \dots, s_n\}.
\]
Each span $s_i$ may correspond to a privacy category such as personally identifiable information, financial information, health-related information, confidential organizational terms, or sensitive text extracted from visual input.

Our goal is to construct a privacy mediation function
\[
\mathcal{M}: x \mapsto \tilde{x},
\]
where $\tilde{x}$ is a sanitized prompt satisfying the following properties:
\begin{enumerate}
    \item privacy-sensitive content in $x$ is hidden, generalized, replaced, or obfuscated in $\tilde{x}$;
    \item the semantic intent of the original prompt is preserved as much as possible;
    \item the downstream model output on $\tilde{x}$ remains useful for the original task.
\end{enumerate}

Given a downstream model $f(\cdot)$, the revised objective is not conventional text classification, but privacy-aware inference:
\[
y = f(\tilde{x}),
\]
where $\tilde{x}$ is generated from $x$ through privacy-preserving prompt mediation before inference.

In some scenarios, an authorized post-processing or secure recovery mechanism may be used. We denote this by
\[
\mathcal{R}(y, K) \mapsto y',
\]
where $K$ represents an access-controlled mapping policy or secure key that enables partial restoration of protected entities only when permitted.

\subsection{Threat Model}

We consider an honest-but-curious or partially trusted deployment environment for LLM/VLM-based agents. The downstream model service, logging components, retrieval pipelines, or external tools may observe raw prompts if no privacy protection is applied before inference. Even if the foundation model itself is not intentionally adversarial, sensitive information may still be exposed through storage, debugging, monitoring, or tool invocation traces.

The attacker is assumed to have access to one or more of the following:
\begin{itemize}
    \item plaintext prompts submitted to the model;
    \item tool arguments generated by an agent;
    \item logged interaction records;
    \item OCR-extracted text or multimodal intermediate representations.
\end{itemize}

The attacker aims to recover or infer private information directly from prompts or indirectly from intermediate system outputs. Therefore, privacy protection should occur before prompt forwarding, rather than only during offline training or after response generation.

We do not assume that all downstream computation is cryptographically secure. Instead, the revised paper focuses on a practical deployment setting in which prompt-level privacy mediation reduces exposure risk while preserving usability and compatibility with existing LLM/VLM agent systems.

\subsection{Design Objective}

The revised framework should balance privacy reduction and task utility. This can be expressed as
\[
\min_{T} \ \mathcal{L}_{privacy}(x,\tilde{x}) + \lambda \mathcal{L}_{utility}(f(x), f(\tilde{x})),
\]
where $T$ denotes the sanitization transformation, $\mathcal{L}_{privacy}$ measures residual privacy exposure, $\mathcal{L}_{utility}$ captures semantic or task degradation, and $\lambda$ controls the privacy--utility trade-off.

This formulation emphasizes that the goal of the revised paper is not merely to mask text, but to preserve enough meaning for correct downstream reasoning after privacy-sensitive content has been protected.
\end{verbatim}
