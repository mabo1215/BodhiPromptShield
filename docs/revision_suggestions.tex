\section{Overall Assessment}
%% ============================================================

\textbf{Recommendation: \textcolor{metaorange}{Major Revision}}

This paper reframes prompt privacy in LLM/VLM agents as a cross-stage
\emph{propagation-control} problem, which carries conceptual novelty.
The systems design is logically structured. However, measured against
top-venue standards (IEEE TIFS, IEEE TDSC, CCS, USENIX Security, IEEE
S\&P, ACL, NeurIPS), the paper has the following critical deficiencies
that must be resolved before acceptance:

\begin{enumerate}[leftmargin=1.5em]
  \item \textbf{Credibility of the self-constructed benchmark (CPPB):}
    All core experiments rely on the authors' own 256-prompt benchmark.
    No complete evaluation on any established public benchmark appears
    in the main text, severely limiting external validity.
  \item \textbf{Insufficient technical depth:}
    The framework is essentially rule-routing + NER composition; the
    margin over Presidio is $\approx 2\%$ PER without statistical
    significance testing.
  \item \textbf{Fundamental experimental design gaps:}
    The key hyperparameters ($\alpha_i, \beta_i$, $\tau$) are not
    tabulated; SPE differences (7--11\%) are not tested for
    statistical significance.
  \item \textbf{Narrative structure problem:}
    A large portion of the main text is dedicated to describing
    what has \emph{not} been completed, which undermines the
    persuasiveness of the existing contributions.
\end{enumerate}

\section{Detailed Weaknesses and Revision Suggestions}
%% ============================================================

\subsection{W1 -- Benchmark Scale and Independence}

\noindent\textbf{Issue:}
CPPB contains only 256 prompts generated from 32 templates
(8 variants each). Intra-template correlation is high, which
explains the implausibly low variance in reported metrics
(e.g., TSR $0.92 \pm 0.00$). The authors simultaneously designed
the framework, constructed the benchmark, and chose the metrics,
creating a clear evaluator bias risk.

\noindent\textbf{Suggested revision (Section~V-B):}
\begin{verbatim}
% Replace the current CPPB-only introduction with:
We evaluate BodhiPromptShield on two complementary
benchmarks. The Controlled Prompt-Privacy Benchmark (CPPB)
targets agent-workflow propagation scenarios. To validate
external generalizability, we additionally report results on
the TAB ECHR anonymization corpus \cite{pilan2022tab} and on
a synthetic i2b2-compatible clinical export
\cite{stubbs2015i2b2}. Table~\ref{tab:cppb_card} describes
the CPPB composition; Table~\ref{tab:external} reports
the main-text external-transfer results.
\end{verbatim}

\noindent\textbf{Additional requirements:}
\begin{itemize}
  \item Report McNemar's test or bootstrap confidence intervals
    for all pairwise comparisons, especially the narrow
    PER gap (9.3\% vs 11.2\% vs 8.1\%).
  \item Introduce a held-out dev/test split within CPPB
    to separate hyperparameter selection from evaluation.
\end{itemize}

\subsection{W2 -- Unclear Novelty Boundary vs.\ Industrial Tools}

\noindent\textbf{Issue:}
The core components (rule recognizers + NER + placeholder
replacement + semantic abstraction + secure mapping table)
substantially overlap with Presidio and similar pipelines.
The paper's best result beats Presidio (+NER) by only
$11.2\% \to 9.3\%$ PER---a gap that may not survive
significance testing.

\noindent\textbf{Suggested addition to Related Work:}
\begin{verbatim}
\paragraph{Differentiation from industrial de-id tools.}
Tools such as Microsoft Presidio and spaCy-based pipelines
provide strong single-boundary PII detection, but they do
not model multi-stage propagation risk across agent
boundaries, nor treat restoration timing as a security
variable. The key distinction of BodhiPromptShield is the
propagation-aware routing objective $\Pi_\pi$ and the
boundary-controlled restoration policy $\rho$, both absent
from document-boundary de-identification systems. We make
this distinction empirically concrete via a paired
McNemar's test ($p < 0.05$) in
Table~\ref{tab:presidio_compare}.
\end{verbatim}

\subsection{W3 -- Ambiguous SPE Definition and Monotonicity}

\noindent\textbf{Issue:}
Stage-wise Propagation Exposure (SPE) is the paper's most
central metric, yet it has no precise operational definition.
Readers cannot determine whether span detection uses string
matching, edit distance, or semantic similarity. Furthermore,
the monotonic SPE decrease across stages
(retrieval $10.7\%$ $\to$ memory $8.9\%$ $\to$ tool $7.1\%$)
is counter-intuitive without explanation.

\noindent\textbf{Suggested formal definition to add before
Table~IV:}
\begin{verbatim}
\begin{definition}[Stage-wise Propagation Exposure]
Let $\mathcal{B}_k$ denote the input context assembled at
agent boundary $k \in \{\text{retrieval}, \text{memory},
\text{tool}\}$ under sanitized prompt $\tilde{x}$.
The SPE at stage $k$ is
\[
  \mathrm{SPE}_k(x, \tilde{x}) =
  \frac{
    \bigl|\{s_i \in E(x) : \mathrm{EM}_\delta(s_i,
    \mathcal{B}_k(\tilde{x})) = 1\}\bigr|
  }{|E(x)|},
\]
where $\mathrm{EM}_\delta$ is a token-overlap indicator
with edit-distance tolerance $\delta \leq 2$.
The macro-averaged SPE over the evaluation set $\mathcal{X}$
is $\frac{1}{|\mathcal{X}|}\sum_x \mathrm{SPE}_k(x,\tilde{x})$.
\end{definition}
The monotonic decrease across stages reflects the serial
pipeline architecture: a span blocked at retrieval cannot
enter memory or tool contexts in the current experiment
design, which assumes non-bypassing pipeline composition.
We discuss the implications of pipeline bypassing in
Section~VII.
\end{verbatim}

\subsection{W5 -- Adversarial Robustness Underreported}

\noindent\textbf{Issue:}
Appendix Table~XXI shows a homoglyph recovery rate of only
42\%, meaning the extractor fails to normalise 58\% of
adversarial homoglyph substitutions. This substantially
contradicts the threat model claim of ``partial'' coverage
for adaptive surface-form evasion. These numbers are buried
in the appendix without analysis.

\noindent\textbf{Suggested revision:}
\begin{itemize}
  \item Elevate adversarial robustness results to a
    subsection in Section~VI.
  \item Explicitly characterise what a 42\% recovery rate
    implies in a real deployment (i.e., over half of
    homoglyph attacks succeed).
  \item Label this as an open problem, not a solved one.
\end{itemize}

\subsection{W6 -- Narrative Bloat and Abstract Format}

\noindent\textbf{Issue:}
Approximately 20\% of the main text describes experiments
that are \emph{not yet complete}, which makes the paper
appear immature. The four-block Abstract format
(Problem/Approach/Key Result/Scope) is non-standard.

\noindent\textbf{Suggested Abstract rewrite:}
\begin{verbatim}
\begin{abstract}
LLM and VLM agents expose user prompts to multiple
internal agent boundaries---retrieval, memory writes, and
tool invocations---creating a cross-stage privacy
propagation risk that single-boundary de-identification
cannot address. We present BodhiPromptShield, a
policy-aware pre-inference mediation framework that detects
privacy-sensitive spans, routes them via typed placeholders,
semantic abstraction, or secure symbolic mapping, and delays
entity restoration to authorized execution boundaries,
treating restoration timing as an explicit security
variable. On the Controlled Prompt-Privacy Benchmark
(CPPB, 256 prompts), BodhiPromptShield reduces stage-wise
propagation exposure to 7.1\% at the tool boundary and
achieves Privacy Exposure Rate 9.3\% with Answer
Consistency 0.94 and Task Success Rate 0.92, outperforming
generic de-identification and remaining competitive with
enterprise staged redaction while offering superior
downstream utility. An initial transfer evaluation on the
TAB anonymization corpus confirms cross-corpus
generalisability of the propagation-control design.
\end{abstract}
\end{verbatim}

\subsection{W7 -- Figure Readability and Information Density}

\noindent\textbf{Issue:}
In Fig.~2, the ``Proposed'' and ``Generic de-id'' curves
are visually indistinguishable (10.7\% vs.\ 14.9\%).
Fig.~3 right panel presents only 3 operating points for a
``Pareto frontier'', which is statistically insufficient.

\noindent\textbf{Suggestions:}
\begin{itemize}
  \item Use a log-scale y-axis or grouped bar chart in
    Fig.~2 to accentuate near-zero differences.
  \item Expand Fig.~3 right panel to at least 6
    threshold values
    ($\tau \in \{0.30, 0.40, 0.50, 0.60, 0.70, 0.80\}$)
    to justify the ``Pareto frontier'' claim.
  \item Submit figures as vector PDF/EPS to avoid
    pixellation in print.
\end{itemize}

%% -----------------------------------------------------------
\subsection{W8 -- Answer Consistency Metric Under-Specified}

\noindent\textbf{Issue:}
AC is defined as $g(y_j, \tilde{y}_j) \in [0,1]$
where $g$ is described only as ``semantic agreement''.
If $g$ is an LLM judge, it introduces another black box.
High AC also does not preclude both outputs being wrong.

\noindent\textbf{Suggestions:}
\begin{itemize}
  \item Specify $g$ precisely (BERTScore, ROUGE-L, human
    annotation, or LLM judge with named model/prompt).
  \item Report absolute downstream task accuracy alongside
    AC to prevent the ``consistently wrong'' pitfall.
\end{itemize}

\section{Minor Issues}
%% ============================================================

\begin{enumerate}[leftmargin=1.5em]
  \item \textbf{Symbol inconsistency:} $S(x)$ is introduced
    in Section~II for the span set and $E(x)$ is reintroduced
    in Section~IV with the same meaning. Unify to one symbol
    throughout.

  \item \textbf{Table IV vs.\ Table V mismatch:}
    Table~IV (SPE) omits the NER-only and enterprise-staged
    redaction baselines that appear in Table~V (PER).
    Align the row sets for direct cross-table comparison.

  \item \textbf{Latency measurement environment undisclosed:}
    Table~VIII lacks CPU/GPU model, concurrency setting, and
    prompt-length distribution. These are required for
    reproducibility.

  \item \textbf{Framework name inconsistency:}
    ``BodhiPromptShield'' and ``PromptShield'' are used
    interchangeably in the appendix. Standardise to one name.

  \item \textbf{Proposition~1 proof gap:}
    The claim ``follows from standard sequential composition''
    requires an explicit independence assumption between span
    sanitization mechanisms. Add a one-line formal
    justification or state the independence assumption
    explicitly.

  \item \textbf{Reference [28] (OWASP 2025):}
    URL-only reference. Add a proper bibliographic entry
    with access date.
\end{enumerate}

\section{Concluding Remarks to Authors}
%% ============================================================

The research question addressed in this paper---cross-stage
privacy propagation in LLM/VLM agents---is timely and
practically relevant. The conceptual contribution of treating
\emph{restoration timing as a security variable} is the
paper's most valuable insight and genuinely advances the
framing beyond single-boundary de-identification.

To meet top-venue standards, the authors must:
(1) \textbf{validate on independent data}---public benchmarks
should appear in the main text, not only as appendix footnotes;
(2) \textbf{quantify claims rigorously}---all key comparisons
need statistical significance testing;
(3) \textbf{write with focus}---all descriptions of
``planned but not done'' work should be consolidated into
a single Limitations section.

In its current form the manuscript reads more like a
technical report than a complete conference or journal paper.
The authors are encouraged to address the above points and
submit to \textbf{IEEE TIFS} as the primary target.
