\subsection*{2. Policy selector $\Pi$ is no longer only a heuristic slogan}

这很好，因为它让读者看到：\textbf{mode selection 不只是“看类别选一下”，而是 policy-dependent decision under deployment constraints。}:contentReference[oaicite:8]{index=8}

\textbf{评价：这是方法部分最重要的增强。}

\section*{Remaining Major Problems}

\subsection*{1. The paper is now cleaner, but still not stronger enough experimentally}

当前最根本的问题已经从“结构不清楚”变成了“实验强度不够”。

你现在的 strongest record-backed claim 很明确：
\begin{itemize}
    \item stage-wise propagation suppression across retrieval/memory/tool；
    \item utility-constrained policy 在 PER/AC/TSR 上取得较平衡结果；
    \item balanced routing 是当前 CPPB 下的 knee point。:contentReference[oaicite:11]{index=11}
\end{itemize}

但顶级审稿人接下来会问的，仍然是：

\begin{quote}
这些结果是否只在 CPPB 这种 controlled synthetic benchmark 上成立？
\end{quote}

而你的文中仍清楚写着：
\begin{itemize}
    \item TAB / i2b2-style transfer 仍是 future validation；
    \item stronger external baselines 仍未做 matched scripts and calibration；
    \item multimodal scripts / OCR manifests / exact backbone logs / multi-seed logs 仍缺。
\end{itemize}

因此，\textbf{文章现在更诚实了，但并没有从根本上变得“更难被拒”。}

\subsection*{2. Enterprise staged redaction is still a dangerous comparison point}

这一版虽然写得更好了，但有一个核心 tension 仍然存在：

\begin{itemize}
    \item Enterprise staged redaction 的 PER = 8.1；
    \item Proposed utility-constrained 的 PER = 9.3；
    \item 也就是说，你的方法并不是 direct exposure 上最优，而是在 utility 上更好。:contentReference[oaicite:13]{index=13}
\end{itemize}

你现在的解释是合理的：你的方法在 exposure comparably low 的前提下，utility 更好，因此 overall operating point 更优。并且 Fig. 3 也试图用 Pareto-style operating regimes 来说明这一点。:contentReference[oaicite:14]{index=14}

但是按顶刊标准，这里仍然不够稳，因为审稿人仍可能说：

\begin{quote}
The proposed method is not the best on direct privacy reduction, and the utility gain is only modest.
\end{quote}

\textbf{建议：}
必须把这一 tension 处理得更主动，而不是只在结果段落顺带解释。

建议在主文结果部分明确加一小段：
\begin{quote}
Why the paper does not optimize PER alone.
\end{quote}
要明确说：
\begin{enumerate}
    \item 企业 staged redaction 实际上是 typed-placeholder-only regime；
    \item 在 exact-value-sensitive 或 context-sensitive tasks 中，它容易牺牲语义结构；
    \item 本文目标不是最小 PER，而是 \textbf{policy-controlled privacy--utility frontier under propagation risk}。
\end{enumerate}

否则，enterprise baseline 会持续成为攻击点。

\subsection*{3. The appendix is better organized, but still reveals the main weakness}

appendix 现在写得更规范，但它实际上也把当前稿件最致命的问题暴露得更清楚了。

根据 appendix 的 artifact table，以下内容仍是 controlled manuscript：

    \item category-wise analysis；

这意味着：

你的主文现在通过“evidentiary hierarchy”把这个问题表达得很诚实，但诚实不等于可以过审。对于顶级 venue，这通常仍会被视为：

\begin{quote}
The paper’s central story is promising, but too much of the interesting evidence remains controlled and not fully reproducible.
\end{quote}

\subsection*{4. Threat-to-evaluation mapping is a good table, but it also highlights missing experiments}

Table I 很好，但它也非常明确地告诉审稿人：

    \item adaptive surface-form evasion：only partial；

你已经把 $\Pi$ 和 $R_{\text{prop}}(\pi,\rho)$ 形式化得更好了，但实验仍然没有完全吃满这部分形式化带来的价值。

否则会让审稿人觉得：
\begin{quote}
