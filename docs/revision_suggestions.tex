\section*{Second-Round Top-Tier Revision Checklist for the Revised Manuscript}

\subsection*{Overall Assessment of the Revised Version}
The revised manuscript is substantially improved. In particular, it now calibrates claims more carefully, explicitly distinguishes controlled evidence from formal guarantees, strengthens the propagation analysis as a systems-level contribution, and provides a more transparent reproducibility scope. These are meaningful advances toward submission quality.

However, the paper is \textbf{still not fully top-tier ready}. The remaining issues are no longer primarily about tone, but about \textbf{evidence density, benchmark hardness, and baseline strength}. The next revision should therefore focus less on general polishing and more on tightening the empirical backbone of the paper.

\subsection*{Highest-Priority Remaining Issues}

\paragraph{1. Move conceptual tables out of the main paper.}
Although Table~I and Table~II are now clearly labeled as conceptual/illustrative, they still occupy prominent space in the main text. For a top-tier submission, this reduces empirical density and may give the impression that the paper still relies too heavily on conceptual framing.

\textbf{Required revision:}
\begin{itemize}
    \item Move Table~I and Table~II to the appendix.
    \item Replace them in the main text with a brief one- or two-sentence summary.
    \item If needed, retain a short forward reference such as:
    \begin{quote}
        Illustrative examples and conceptual operating regimes are provided in the appendix for reader orientation.
    \end{quote}
\end{itemize}

\paragraph{2. Harden the CPPB benchmark description with explicit sample accounting.}
The current CPPB description is much improved conceptually, but it still lacks the hard benchmark details expected by top-tier reviewers. The manuscript should report exact counts rather than only benchmark design principles.

\textbf{Required revision:}
\begin{itemize}
    \item Add the total number of prompts in CPPB.
    \item Add the number of prompts in each subset:
    \begin{itemize}
        \item essential-privacy,
        \item incidental-privacy,
        \item multimodal/OCR-mediated,
        \item hard-case subsets.
    \end{itemize}
    \item Add category-level counts or percentages for:
    \begin{itemize}
        \item person identifiers,
        \item financial identifiers,
        \item medical entities,
        \item addresses/locations,
        \item organization/project terms,
        \item context-dependent spans,
        \item OCR-derived spans.
    \end{itemize}
    \item Add prompt-source composition:
    \begin{itemize}
        \item direct user requests,
        \item document-oriented prompts,
        \item retrieval-style prompts,
        \item tool-oriented prompts,
        \item multimodal prompts.
    \end{itemize}
    \item Add the number of prompt templates and generated variants.
    \item State clearly whether each raw prompt is paired with one mediated version per baseline under the same downstream setting.
\end{itemize}

\paragraph{3. Add a benchmark reproducibility map with script-level linkage.}
The reproducibility language is now more transparent, but top-tier standards require a more operational mapping between manuscript artifacts and released scripts.

\textbf{Required revision:}
\begin{itemize}
    \item Add a compact table in the appendix of the form:
    \begin{center}
    Artifact $\rightarrow$ Script/CSV $\rightarrow$ Reproducible in current repository? $\rightarrow$ Notes
    \end{center}
    \item Include all main tables and figures.
    \item Explicitly distinguish:
    \begin{enumerate}
        \item fully regenerated artifacts,
        \item CSV-backed artifacts,
        \item controlled manuscript artifacts not yet end-to-end regenerated.
    \end{enumerate}
    \item Add one command summary for rebuilding the reproducible subset.
\end{itemize}

\paragraph{4. Add at least one stronger deployment-relevant baseline as an actual experiment.}
The revised manuscript honestly states that stronger baselines such as Presidio-class systems or prompt-native privacy tools are not yet included. This is good transparency, but it is still a weakness for top-tier review.

\textbf{Required revision:}
\begin{itemize}
    \item Add one stronger baseline that is actually benchmarked, not only discussed.
    \item The best practical option is a staged enterprise-style baseline:
    \begin{enumerate}
        \item regex/pattern matching,
        \item NER,
        \item category-aware replacement,
        \item no semantic abstraction,
        \item no controlled boundary restoration.
    \end{enumerate}
    \item If feasible, add one external practical comparator such as:
    \begin{itemize}
        \item Microsoft Presidio-style pipeline,
        \item prompted LLM zero-shot de-identification,
        \item strong contextual redaction pipeline.
    \end{itemize}
    \item In the paper, explain why this new baseline is more representative of realistic deployment than the current lightweight comparators.
\end{itemize}

\paragraph{5. Make propagation suppression the undisputed central contribution.}
The revised manuscript greatly strengthens the propagation experiment, but the paper can still push this further. The propagation result is the most distinctive systems contribution and should dominate the narrative.

\textbf{Required revision:}
\begin{itemize}
    \item Mention propagation suppression explicitly in the title candidate discussion, abstract, introduction, and conclusion.
    \item In the introduction, present the core problem as:
    \begin{quote}
        prompt privacy risk is amplified by propagation across retrieval, memory, and tool boundaries.
    \end{quote}
    \item In the abstract, state the stage-wise propagation result directly.
    \item In the conclusion, restate propagation suppression as the strongest validated finding.
    \item Consider moving the propagation experiment earlier within the results section.
\end{itemize}

\subsection*{Section-by-Section Final Tightening}

\paragraph{Abstract}
The abstract is now more careful, but it can still be sharpened.

\textbf{Required revision:}
\begin{itemize}
    \item Move the propagation claim earlier.
    \item Explicitly state that the evaluation is based on a controlled benchmark.
    \item Reduce generic phrasing such as ``model-agnostic'' unless the cross-model scope is precisely qualified.
    \item End with a narrower conclusion centered on controlled evidence for reduced cross-stage propagation and preserved task behavior.
\end{itemize}

\paragraph{Introduction}
The introduction is stronger than before, especially in claim calibration.

\textbf{Required revision:}
\begin{itemize}
    \item Further sharpen the gap against enterprise redaction middleware and ordinary de-identification pipelines.
    \item Emphasize earlier that the main novelty is \emph{pre-inference mediation under propagation risk}, not merely prompt sanitization.
    \item Shorten any residual general discussion that does not directly support the central propagation narrative.
\end{itemize}

\paragraph{Related Work}
This section is adequate, but still somewhat descriptive.

\textbf{Required revision:}
\begin{itemize}
    \item Add a sharper contrast paragraph focused on stage of protection:
    \begin{itemize}
        \item training-time protection,
        \item stored-text de-identification,
        \item inference-time prompt security,
        \item agent-boundary mediation.
    \end{itemize}
    \item Make clearer why existing de-identification pipelines are insufficient once prompt content can propagate into memory, retrieval, and tool arguments.
\end{itemize}

\paragraph{Method}
The method section is clear, but still slightly overloaded by explanatory material.

\textbf{Required revision:}
\begin{itemize}
    \item Remove conceptual tables from this section.
    \item Add one explicit paragraph clarifying which components are:
    \begin{enumerate}
        \item currently implemented,
        \item experimentally validated,
        \item proposed as deployment guidance only.
    \end{enumerate}
    \item Make the policy selector $\Pi$ slightly more operational:
    what features or signals does it practically consume?
\end{itemize}

\paragraph{Experiments}
This section improved substantially, but it is still the main bottleneck.

\textbf{Required revision:}
\begin{itemize}
    \item Add exact CPPB counts and distributions.
    \item Add one stronger benchmarked baseline.
    \item Add an appendix reproducibility table linking artifacts to scripts.
    \item If possible, add one small repeated-run or robustness analysis to strengthen the credibility of the operating-point figures.
\end{itemize}

\paragraph{Results}
The revised results section is one of the strongest parts of the paper.

\textbf{Required revision:}
\begin{itemize}
    \item Move the propagation subsection earlier, or explicitly announce it as the central systems result.
    \item Add one short takeaway sentence at the end of each subsection.
    \item Trim any explanatory repetition once the key claim has been established.
\end{itemize}

\paragraph{Limitations}
The limitations section is now mature and top-tier in tone.

\textbf{Required revision:}
\begin{itemize}
    \item Keep the current honesty.
    \item Add one sentence clarifying that the present evidence should be interpreted as controlled systems validation rather than a deployment certification.
    \item Explicitly mention that the absence of stronger matched baselines remains part of the present scope limitation.
\end{itemize}

\paragraph{Conclusion}
The conclusion is much improved, but can still be more memorable.

\textbf{Required revision:}
\begin{itemize}
    \item Restate only the strongest validated result.
    \item Make propagation suppression the centerpiece of the concluding claim.
    \item Avoid broad wording such as universal middleware compatibility unless tightly qualified.
    \item End with a concrete next milestone: stronger baselines, fuller CPPB release, and broader external validation.
\end{itemize}

\subsection*{Recommended Final Reframing Sentence}
A strong manuscript-level calibration sentence would be:

\begin{quote}
This paper should be interpreted as controlled systems evidence that policy-aware pre-inference mediation can reduce privacy propagation across agent boundaries while preserving useful downstream behavior, rather than as a universal privacy guarantee for all LLM/VLM deployments.
\end{quote}

\subsection*{Recommended Final Submission Priorities}
If revision time is limited, the final priorities should be:

\begin{enumerate}
    \item move conceptual tables to the appendix;
    \item add exact CPPB counts, distributions, and source composition;
    \item add an artifact-to-script reproducibility map;
    \item benchmark one stronger deployment-relevant baseline;
    \item promote propagation suppression to the dominant narrative in abstract, introduction, results, and conclusion.
\end{enumerate}
