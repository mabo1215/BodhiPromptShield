\section*{Stage 9 Experimental Review and Recommendations for Cursor}
\label{sec:stage9_experimental_review}

\subsection*{Overall Experimental Assessment}
The current manuscript already has a coherent experimental skeleton. In particular, the paper now includes:
\begin{itemize}
    \item a controlled benchmark definition (CPPB) in Section V-B;
    \item a paired-comparison protocol in Section V-C;
    \item representative operational baselines in Section V-D;
    \item privacy, utility, and latency metrics in Sections V-E and VI;
    \item ablation and deployment-oriented discussion in Tables III--VI and Section VI.
\end{itemize}

This is a strong improvement over earlier drafts, and the experimental narrative is now aligned with the prompt privacy problem rather than the legacy classifier storyline. This is visible in the current PDF, especially in the CPPB construction on page 5, the PER / AC / TSR / UPR tables on pages 6--7, and the deployment interpretation on page 7. :contentReference[oaicite:0]{index=0}

\subsection*{Direct Answer to the Question ``What Experiments Still Need to Be Added?''}
Yes, the paper still needs additional experiments if the goal is to make it more convincing, especially for a stronger venue. The current results are coherent, but they are still relatively compact and prototype-like. The main missing pieces are:

\begin{enumerate}
    \item cross-model validation;
    \item category-wise detection and protection breakdown;
    \item stronger multimodal evaluation;
    \item robustness / stress-test experiments;
    \item restoration-boundary experiments;
    \item policy-sensitivity experiments;
    \item longer or multi-step agent pipeline experiments.
\end{enumerate}

\subsection*{Most Important Missing Experiment 1: Cross-Model Validation}
At present, the paper evaluates the framework under a controlled protocol, but it does not yet show whether the mediation behavior is stable across multiple downstream model backends. This is a major gap if the paper claims model-agnostic deployment.

Please add a new experiment subsection in Section V or VI:

\begin{verbatim}
\subsection{Cross-Model Validation}
To test whether the mediation framework is robust across downstream model backends, we evaluate the same sanitized prompts on multiple LLM/VLM systems under matched prompt settings. The goal is not to exhaustively benchmark all model families, but to assess whether the privacy--utility trends remain stable across representative downstream models with different reasoning and formatting behaviors.
\end{verbatim}

Recommended comparison:
\begin{itemize}
    \item one stronger general LLM;
    \item one smaller or lighter LLM;
    \item if possible, one multimodal model for the image-assisted subset.
\end{itemize}

Recommended table:
\begin{verbatim}
Table X. Cross-model utility retention under the same mediation policy.
\end{verbatim}

Metrics:
\begin{itemize}
    \item AC;
    \item TSR;
    \item optionally UPR.
\end{itemize}

\subsection*{Why This Experiment Is Important}
Your abstract and conclusion say the method is model-agnostic and compatible with text-only and multimodal agents. That claim is reasonable, but it is currently supported mostly by architectural argument rather than breadth of empirical validation. A cross-model comparison would make that claim much more credible. This is especially important because the current evaluation protocol on pages 5--7 is still concentrated around a single controlled setup. :contentReference[oaicite:1]{index=1}

\subsection*{Most Important Missing Experiment 2: Category-Wise Breakdown}
Right now, PER is only reported at the overall method level in Table III. That is useful, but not enough. Since your framework explicitly handles multiple privacy categories, the paper should report category-level behavior.

Please add a new subsection:

\begin{verbatim}
\subsection{Category-Wise Privacy Protection Analysis}
We further break down privacy mediation performance by entity category, including person names, addresses, identifiers, financial spans, medical spans, and OCR-derived visual spans. This analysis clarifies which categories are well covered by rule-based recognizers, which benefit from contextual judgment, and which remain challenging under the current framework.
\end{verbatim}

Recommended table:
\begin{verbatim}
Table X. Category-wise Sensitive Span F1 and PER under the proposed framework.
\end{verbatim}

Suggested columns:
\begin{itemize}
    \item Category;
    \item Precision;
    \item Recall;
    \item F1;
    \item PER after mediation.
\end{itemize}

\subsection*{Why This Experiment Is Important}
Your framework in Section IV-B explicitly distinguishes structured entities, contextual entities, and OCR/VLM-derived spans. Reviewers will expect category-level evidence that these components matter differently. Without this table, the paper still looks slightly too aggregated. The need for this is already implied by the mixed categories listed in pages 3--5 and by the failure cases on page 7. :contentReference[oaicite:2]{index=2}

\subsection*{Most Important Missing Experiment 3: Multimodal Evaluation Must Be Stronger}
The paper includes a multimodal extension in Section IV-H and mentions OCR-derived visual spans, but the actual experimental evidence for multimodal performance is still thin. The multimodal claim is currently more architectural than empirical.

Please add a dedicated multimodal experiment subsection:

\begin{verbatim}
\subsection{Multimodal Prompt Mediation Evaluation}
We evaluate the multimodal branch of the framework on prompts containing screenshots, invoices, forms, or medical-document snippets with OCR-derived private content. The purpose is to test whether visual-entity extraction and sanitization remain effective when privacy-bearing information originates from image content rather than typed text.
\end{verbatim}

Recommended evaluation tasks:
\begin{itemize}
    \item invoice or receipt summarization;
    \item ID-card or form description;
    \item medical image report summarization with OCR text;
    \item reimbursement or document-understanding prompts.
\end{itemize}

Recommended metrics:
\begin{itemize}
    \item visual-span Precision / Recall / F1;
    \item PER on OCR-derived spans;
    \item AC on multimodal tasks.
\end{itemize}

Recommended table:
\begin{verbatim}
Table X. Multimodal privacy mediation results on OCR-derived entity spans.
\end{verbatim}

\subsection*{Why This Experiment Is Important}
At present, the multimodal claim is plausible, but top reviewers will likely say: ``the paper says VLM, but where is the actual multimodal evidence?'' The current PDF includes the multimodal formulation on page 4 and a multimodal example in the appendix, but the empirical section does not yet match that scope strongly enough. :contentReference[oaicite:3]{index=3}

\subsection*{Important Missing Experiment 4: Policy Sensitivity Analysis}
The policy selector $\Pi$ is central to the paper, but it is not yet stress-tested enough. Table V helps, but it still reports only three operating modes. A stronger experiment would show how the system behaves as policy aggressiveness changes.

Please add:

\begin{verbatim}
\subsection{Policy Sensitivity Analysis}
To study how policy strength affects deployment behavior, we vary the mediation policy from conservative to balanced to aggressive settings and measure the resulting changes in PER, AC, TSR, and latency. This experiment tests whether the policy selector $\Pi$ provides a stable and interpretable operating frontier rather than only isolated configuration points.
\end{verbatim}

Recommended outputs:
\begin{itemize}
    \item one table with 4 metrics;
    \item optionally one figure: PER vs AC or PER vs UPR;
    \item optionally latency vs policy level.
\end{itemize}

This can reuse your current conceptual trade-off, but now with empirical points added.

\subsection*{Why This Experiment Is Important}
Section IV-E says $\Pi$ is the operational core of the framework, and Table V on page 7 already points in that direction. :contentReference[oaicite:4]{index=4}  
However, to make $\Pi$ feel truly central, you need one experiment showing that changing policy strength leads to predictable exposure--utility trade-offs.

\subsection*{Important Missing Experiment 5: Authorized Restoration Boundary Test}
Your framework includes symbolic mapping and authorized restoration, but there is still no explicit experiment showing when restoration helps and what risk it introduces.

Please add:

\begin{verbatim}
\subsection{Authorized Restoration Boundary Analysis}
We evaluate symbolic mapping with and without authorized restoration at the final execution boundary. The goal is to measure whether exact-value restoration improves downstream task completion when tools require precise identifiers, and to quantify how much exposure remains avoided compared with raw prompt forwarding.
\end{verbatim}

Recommended comparison:
\begin{itemize}
    \item raw prompt;
    \item symbolic mapping without restoration;
    \item symbolic mapping with restoration only at tool boundary.
\end{itemize}

Recommended metrics:
\begin{itemize}
    \item TSR;
    \item PER;
    \item optionally a new metric: restored-boundary exposure count.
\end{itemize}

\subsection*{Why This Experiment Is Important}
This experiment would strongly support your claim that restoration should happen only at the final boundary rather than earlier in the chain. That idea is already present in Sections IV and VI, but it is not yet experimentally isolated. The argument around tool-boundary restoration on page 6 and in Table V would become much stronger if directly tested. :contentReference[oaicite:5]{index=5}

\subsection*{Important Missing Experiment 6: Robustness to Hard Cases}
The paper already lists failure modes in Section VI-G, but these are discussed qualitatively. It would help to convert them into a small robustness experiment.

Please add:

\begin{verbatim}
\subsection{Robustness Under Hard Privacy Cases}
We construct a hard subset including context-dependent entities, nested identifiers, partially formatted financial strings, long enterprise-specific names, OCR-noisy spans, and mixed text-image prompts. This subset is used to stress-test the limits of privacy entity extraction and policy-aware sanitization.
\end{verbatim}

Recommended metrics:
\begin{itemize}
    \item category-wise F1;
    \item PER;
    \item failure-rate percentage.
\end{itemize}

This can be a small table rather than a full section if space is tight.

\subsection*{Why This Experiment Is Important}
Right now, your failure case discussion on page 7 is good, but it is still descriptive. :contentReference[oaicite:6]{index=6}  
Turning at least part of it into a measured hard-case subset would make the paper look more rigorous and less template-like.

\subsection*{Important Missing Experiment 7: Longer Agent-Chain Evaluation}
The paper claims agent-level integration with retrieval, memory, and tools, but the current experiments still feel mostly single-prompt or short-pipeline oriented.

Please add:

\begin{verbatim}
\subsection{Multi-Step Agent Propagation Evaluation}
We simulate a multi-step agent setting in which a protected prompt is passed through retrieval, memory update, and tool argument construction. We then measure whether early mediation reduces downstream propagation of raw sensitive spans across intermediate agent states.
\end{verbatim}

Recommended metric:
\begin{itemize}
    \item number of raw sensitive-span appearances across intermediate states;
    \item final TSR;
    \item PER at each pipeline stage.
\end{itemize}

Recommended table:
\begin{verbatim}
Table X. Propagation of raw sensitive spans across agent pipeline stages.
\end{verbatim}

\subsection*{Why This Experiment Is Important}
Your threat model and framework strongly emphasize propagation risk through memory, retrieval, and tools. That is one of the paper's best ideas. But the current evidence does not yet isolate propagation across stages explicitly. The workflow figure on page 4 clearly motivates this experiment, so adding it would tighten the paper substantially. :contentReference[oaicite:7]{index=7}

\subsection*{Bottom-Line Review Judgment on Experiments}
The current experiments are \textbf{coherent but still not broad enough}. They are sufficient for:
\begin{itemize}
    \item a strong prototype paper;
    \item an application/systems-oriented submission;
    \item a controlled proof-of-concept framing.
\end{itemize}

They are \textbf{not yet sufficient} for a highly competitive venue if reviewers expect:
\begin{itemize}
    \item broader model coverage;
    \item stronger multimodal evidence;
    \item category-level transparency;
    \item restoration-specific validation;
    \item propagation-aware agent measurements.
\end{itemize}
