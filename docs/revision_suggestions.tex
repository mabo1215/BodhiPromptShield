\section*{Reviewer Report for IEEE Transactions on Information Forensics and Security (TIFS)}

\subsection*{Overall Assessment}

This manuscript is now substantially stronger than in earlier cycles. The paper presents a coherent propagation-control framing for prompt privacy mediation, the CPPB evidence backbone is stable, and the repository now includes materially better external and threat-facing support: executed TAB and AI4Privacy public transfer slices, a public relabeled clinical supporting route, a stronger Presidio-class comparison surface, an executed adversarial probe suite with confusable hardening, and a small executed context-inference attack suite.

The paper is therefore no longer limited by missing scaffolds or purely manuscript-authored placeholders. The remaining concerns are more specific and harder: the security story is still only partially closed under adaptive attacks, the external-baseline family is still not strong enough for an information-security venue at the current main-text level, and the formal privacy scope remains intentionally narrow. These issues are now evidence-bounded rather than prose-bounded.

\textbf{Recommendation: Major Revision}

\subsection*{Primary Strengths}

\begin{itemize}
\item The manuscript now tells one consistent story: prompt privacy is treated as a propagation-control problem across retrieval, memory, and tool boundaries rather than as single-boundary redaction.
\item The core CPPB backbone is well organized and reproducibility-aware, with deterministic accounting, split surfaces, multi-seed summaries, leave-template-out evaluation, and supporting artifact maps.
\item External validation is materially better than before: TAB and AI4Privacy provide public text-transfer evidence, PhysioNet-relabeled notes provide a public clinical supporting path, and the OCR route is executed on CORD, FUNSD, and SROIE under one declared stack.
\item The paper is now more honest about its limits. In particular, the adversarial suite and the context-inference suite expose real residual risk rather than over-claiming robustness or privacy closure.
\end{itemize}

\subsection*{Remaining Concerns}

\paragraph{1. Adaptive attack coverage remains only partial.}
The new executed adversarial and context-inference suites are valuable, but they also make the remaining weakness more explicit. Unicode/confusable hardening reduces homoglyph exposure, yet residual exposure is still large, and paraphrase, mixed-language, and restoration-trigger probes remain only partially mitigated. Likewise, the new context-inference suite moves that threat from untested to executed, but sanitized prompt history still permits non-trivial attribute inference.

This is now the most important actionable gap for TIFS. The issue is no longer whether the paper acknowledges adaptive attacks; it does. The issue is whether the current defense surface is strong enough to support the paper's security framing.

\textbf{Priority for the next cycle:}
\begin{itemize}
\item Expand the attack surface from single-prompt probes to at least one multi-turn or multi-stage adaptive scenario.
\item Strengthen surface-form normalization beyond the current one-to-one confusable folding.
\item Keep the paper's tone evidence-bounded unless the new attack results materially improve.
\end{itemize}

\paragraph{2. The strongest external baseline family is still incomplete in the main evidentiary story.}
The repository now contains useful baseline support, including Presidio-class comparisons and multiple local zero-shot pilots. However, the main reviewer concern is no longer ``are there any stronger baselines at all?'' but rather ``is the strongest modern comparison family strong enough for this venue?''

The paper still lacks a broader main-text comparison against modern semantic baselines or stronger named third-party systems on the central CPPB surface. The executed public semantic pilots help, but they remain fragmented across transfer slices rather than forming one convincing, high-strength baseline family aligned to the main claim.

\textbf{Priority for the next cycle:}
\begin{itemize}
\item Promote one stronger semantic or named baseline into a more central comparison role if a fair released runtime can be bundled.
\item If that is not feasible, tighten the paper's comparative claims further so that the current baseline family is presented as practical but not exhaustive.
\end{itemize}

\paragraph{3. The formal privacy scope remains intentionally narrow and should stay explicitly scoped.}
The current span-level LDP analysis is useful as a narrow analytical component, but it does not provide a global security guarantee for the deployed system. The manuscript is already fairly honest about this. The remaining risk is not that the paper lacks any formal discussion, but that reviewers may still read the paper as making a stronger privacy claim than the analysis supports.

\textbf{Priority for the next cycle:}
\begin{itemize}
\item Preserve the current narrow wording unless a stronger analytical result is actually added.
\item Avoid any phrasing that could be read as system-level formal privacy assurance.
\end{itemize}

\paragraph{4. Exact disclosure and portability metadata are improved but still ceiling-limited.}
The paper now distinguishes missing schema from missing filled records, which is the correct move. Even so, some cross-model and multimodal slices still lack the filled disclosure records needed for exact regeneration.

\textbf{Priority for the next cycle:}
\begin{itemize}
\item Continue to present these slices as bounded supporting evidence rather than exact-regeneration artifacts.
\item Only promote them further if releasable filled records become available.
\end{itemize}

\subsection*{Recommended Next-Cycle Priorities}

\begin{enumerate}
\item \textbf{Highest priority:} strengthen adaptive-attack evaluation, especially multi-turn or multi-stage attacks and stronger surface-form normalization.
\item \textbf{Second priority:} improve the strength or centrality of the strongest modern baseline family, or further narrow the paper's comparison claims if stronger executed baselines cannot be released.
\item \textbf{Third priority:} preserve strict honesty around formal privacy scope and exact disclosure; do not broaden claims beyond what the current evidence supports.
\end{enumerate}

\subsection*{Bottom Line}

The manuscript is now much closer to a stable submission than in earlier rounds, and several previously serious concerns have already been reduced by real repository-backed evidence. The remaining work is concentrated in three areas: adaptive attack robustness, strength of the modern baseline family, and disciplined scoping of formal privacy claims. These are difficult but clearly defined problems, making the next revision cycle more focused than the earlier ones.
